# 作業条件

- 現在のディレクトリはこのタスク専用の git worktree で、作業ブランチ `{{branch}}`（ベース: `{{base_branch}}`）がすでにチェックアウト済みです。ブランチの切り替えや新規作成はしないでください。
- リポジトリ設定に応じて、必要な環境ファイル（`.env` 等）のコピーや依存インストール等のセットアップが実行済みの場合があります。まず既存の状態でテスト等を試し、不足していれば必要な分だけ補ってください。
- リポジトリの CLAUDE.md・コーディング規約・コミット規約（Conventional Commits 等）があればそれに従ってください。
- コード変更を伴うタスクの場合:
  - 実装後、リポジトリの標準的な方法でテスト・lint を実行し、通ることを確認してください。
  - コミットを作成し、`git push -u origin {{branch}}` で push し、`gh pr create --draft` で {{base_branch}} 向けの Pull Request を作成してください。PR 本文には Notion ページへのリンク（{{page_url}}）を含めてください。
  - PR はドラフトで作成してください。
- 調査・分析・情報収集のみで完結し、コード変更が不要なタスクの場合:
  - リポジトリへの変更・コミット・PR 作成は不要です。調査結果や結論を完了報告の `summary` にまとめて報告してください。

# 完了報告（必須）

作業の最後に、結果を以下の JSON 形式で `{{result_file}}` に書き込んでください（成否にかかわらず必ず書くこと）:

```json
{
  "status": "success",
  "pr_url": "https://github.com/org/repo/pull/123",
  "summary": "何をしたかの簡潔な説明（日本語、3行以内）"
}
```

- 実装して PR を作成できた場合: `status` は `"success"`、`pr_url` に PR の URL。
- コード変更が不要なタスクを完了した場合: `status` は `"success"`、`pr_url` は省略し、`summary` に実施内容と結論を書いてください。
- 要件が曖昧・判断が必要・権限不足など、**人間の回答があれば作業を続行できる**場合: `status` は `"needs_info"` とし、`question` に確認したいことを具体的に書いてください（複数あれば箇条書き）。質問はそのまま Notion コメントとして人間に届き、回答が付くと作業が自動で再開されます。中途半端な実装・push・PR 作成はしないでください。

```json
{
  "status": "needs_info",
  "question": "確認したいことを具体的に（複数あれば箇条書き）"
}
```

- 継続不能な問題があった場合: `status` は `"failure"`、`pr_url` は省略し、`summary` に理由と人間への引き継ぎ事項を書いてください。中途半端な push や PR 作成はしないでください。
- `failure` と `needs_info` の使い分け: 人間から回答をもらっても続行できない技術的問題（環境が壊れている・テスト失敗が解消できない等）は `failure`、人間の判断・情報があれば解決するもの（仕様の選択・要件の確認・権限付与の依頼等）は `needs_info` にしてください。
